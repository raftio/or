# RFC-005: KPI & Outcome Definition Schema

**Status:** Draft  
**Summary:** Defines the schema for KPI definitions and outcome signals: metric type, aggregation, link to release artifacts, and post-release insight format.

---

## 1. Overview

Outcomes are measurable results of releases. This RFC defines how KPIs and outcome signals are described and linked to release artifacts, and how post-release insights are formatted. The Outcome Feedback Loop (RFC-019) uses this schema for collection, attribution, and reporting.

**In scope:**

- Metric types and aggregation (e.g. count, rate, percentile).
- Link to release artifacts (release id, version, tag).
- Post-release insight format (what is stored or exposed after measurement).

**Out of scope:**

- How metrics are collected from external systems (RFC-019).
- Control Plane API for reading/writing KPIs (RFC-006).

---

## 2. Schema

### 2.1 Metric types and aggregation

- **Types:** e.g. `count`, `rate`, `percentage`, `latency`, `throughput`, `custom`. Optional unit (e.g. ms, req/s).
- **Aggregation:** How raw data is aggregated for the outcome: e.g. `sum`, `avg`, `p50`/`p95`/`p99`, `min`/`max`, or custom (e.g. “conversion rate” as derived).
- **Window:** Optional time window for aggregation (e.g. last 7 days post-release).

KPI definitions reference a metric type and aggregation so that the same definition can be applied across releases.

### 2.2 Link to release artifacts

- **Release identifier:** Stable id for a release (e.g. version tag, deployment id, or `{repo}:{tag}` for multi-repo). Outcome signals are attributed to one or more releases.
- **Artifact references:** Optional list of artifacts (e.g. build ids, manifest hashes) that belong to this release for traceability.
- **Ticket/bundle link:** Optional link from outcome back to ticket(s) or bundle(s) that contributed to the release, for “outcome per ticket” or “outcome per initiative” views.

### 2.3 Post-release insight format

Post-release insights are the stored or exported result of measuring outcomes:

- **Outcome record:** Release id, KPI id, computed value, window, timestamp.
- **Comparison:** Optional comparison to previous release or baseline (e.g. delta, percent change).
- **Attribution:** Optional link to tickets/features that shipped in this release (for product insights).

Format is suitable for storage in the Control Plane and for display in UI or export (e.g. for dashboards).
