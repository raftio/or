# RFC-013: IDE / Agent Integration

**Status:** Draft  
**Summary:** Defines the interface with VSCode, Cursor, JetBrains: how IDE/agent receives execution bundle, submits evidence, and syncs context.

---

## 1. Overview

Developers and AI agents work in IDEs (VSCode, Cursor, JetBrains). Orqestra supplies the execution bundle and can receive lightweight evidence or context updates from the IDE. This RFC defines the integration: receive bundle, submit evidence/sync context, and options for each IDE family.

**In scope:**

- How the IDE/agent receives the execution bundle (API, extension).
- How the IDE/agent submits evidence or syncs context (e.g. “task started”, “tests run locally”).
- VSCode/Cursor vs JetBrains (extension vs API or plugin).

**Out of scope:**

- Control Plane API details (RFC-006) — only the usage pattern from IDE perspective.
- Execution bundle schema (RFC-002) — consumed as-is.

---

## 2. Interface

### 2.1 Receive execution bundle

- **Trigger:** User (or agent) selects a ticket or opens “Get bundle for ticket”. IDE calls Control Plane API (RFC-006): e.g. `GET /bundles?ticket_id=PROJ-123` or `POST /context` with ticket id.
- **Response:** Execution bundle (RFC-002): tasks, acceptance criteria refs, context. IDE displays it in a sidebar, panel, or agent context so the developer/agent can implement against it.
- **Refresh:** When spec or ticket is updated, a new bundle version may exist. IDE can poll or subscribe (e.g. “bundle updated” notification) to refresh; optional “pin to version” to avoid mid-session changes.

### 2.2 Submit evidence / sync context

- **Evidence:** IDE or local test run may produce evidence (e.g. “local tests passed”, coverage snapshot). Optional: send a minimal evidence payload (RFC-004) to the Control Plane so that PR Intelligence can show “local checks” in addition to CI evidence. Format same as RFC-004; source field can indicate “ide” or “local”.
- **Context sync:** Optional: IDE reports “task X started” or “focus on task Y” for workflow orchestration (RFC-018) or for analytics. Lightweight events; no large payloads.
- **No code push:** IDE does not push code to Git; that remains the developer’s action. Orqestra only receives metadata and evidence.

### 2.3 VSCode / Cursor / JetBrains

- **VSCode / Cursor:** Extension (Orqestra extension) that uses Control Plane API: fetch bundle, show in webview or tree view; optional “Send evidence” or “Sync status”. Can use existing auth (e.g. API key in settings).
- **JetBrains:** Plugin with same capabilities: bundle view, optional evidence submit. Plugin uses same HTTP API as extension.
- **Agent (e.g. Cursor agent):** Agent can call the same API (with user’s credentials or workspace config) to load bundle into context and optionally report progress or evidence. No IDE-specific UI required; API-only integration is sufficient for agents.
