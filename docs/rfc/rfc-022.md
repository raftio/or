# RFC-022: AI Chat Agent

**Status:** Accepted
**Created:** 2026-02-28

## Problem

Users working with Orca's control plane need a conversational interface to ask questions about their workspace data — execution bundles, task statuses, evidence results, and integrations. Currently, understanding workspace state requires navigating multiple pages. A chat agent would let users get contextual answers in natural language.

## Solution

A workspace-aware AI chat agent that:

1. Streams responses via Server-Sent Events (SSE) using the Vercel AI SDK
2. Injects workspace context (recent bundles, evidence) into the system prompt
3. Persists conversations in PostgreSQL for history and continuity
4. Lives in a standalone `@orca/agent` package for independent scaling and swappability

## Architecture

### Package Separation

The agent logic is extracted into `packages/agent` (`@orca/agent`) to decouple AI concerns from the HTTP layer:

- `@orca/agent` — contract, provider implementations (OpenAI, Anthropic, stub), system prompt builder, factory
- `apps/api` — HTTP routes, persistence (chat-store), workspace context injection (chat-context)
- `apps/web` — Chat UI page using `useChat` from `ai/react`

### Data Model

Two new tables:

- `workspace_chat_conversations` — conversation metadata (workspace, user, title, timestamps)
- `workspace_chat_messages` — individual messages (conversation FK, role, content, timestamp)

Messages cascade-delete when a conversation is deleted.

### Provider Strategy

Same pattern as the AI decomposer (RFC-006):

- Env-based config: `AI_CHAT_PROVIDER` (stub/openai/anthropic) + `AI_CHAT_MODEL`
- Per-workspace override via `workspace_integrations` table
- Automatic fallback to stub when no API key is available

## API Endpoints

| Method | Path | Description |
|--------|------|-------------|
| POST | `/v1/workspaces/:id/chat` | Send messages, receive streaming response |
| GET | `/v1/workspaces/:id/chat/conversations` | List conversations |
| GET | `/v1/workspaces/:id/chat/conversations/:id` | Get conversation with messages |
| DELETE | `/v1/workspaces/:id/chat/conversations/:id` | Delete conversation |

All endpoints require authentication and workspace membership.

### Streaming Chat (POST)

Request body:

```json
{
  "messages": [
    { "role": "user", "content": "What bundles do I have for PROJ-42?" }
  ],
  "conversationId": "optional-uuid"
}
```

Response: SSE data stream (Vercel AI SDK format). The `X-Conversation-Id` header contains the conversation ID for persistence.

## Configuration

| Variable | Default | Description |
|----------|---------|-------------|
| `AI_CHAT_PROVIDER` | `stub` | Chat AI provider: `stub`, `openai`, or `anthropic` |
| `AI_CHAT_MODEL` | Provider default | Override model name |
| `OPENAI_API_KEY` | — | Required for `openai` provider |
| `ANTHROPIC_API_KEY` | — | Required for `anthropic` provider |

Default models: `gpt-4o-mini` (OpenAI), `claude-sonnet-4-20250514` (Anthropic).

## System Prompt Design

The system prompt has two parts:

1. **Base persona** — Orca Assistant: engineering assistant that understands bundles, evidence, tasks, and acceptance criteria
2. **Workspace context** — Dynamically injected summary of recent bundles (last 10) and evidence (last 10), kept under ~2000 tokens

The context is rebuilt per request to reflect the latest workspace state.

## Frontend

The chat page at `/chat` features:

- Left panel: conversation list with create/delete
- Main area: message thread with streaming, auto-scroll
- Input bar: textarea with Enter-to-send, Shift+Enter for newline
- Uses `useChat` from `ai/react` for streaming integration
